table 부모태그/ 자식태그 caption 표이름 thead(머릿말), tbody(본문), tfoot(합계 등), tr표 행태그/ 손자태그 th 1행내용 td 다음행
흔히쓰는 표만들기. tr로 감쌀때마다 1행씩 늘어남. 그 tr의 자식태그로 보통 제일 윗줄 내용설명칸을 맡음. 그후 행부턴 td사용
스타일로 선을 충분히 세련되게 꾸밀 수 있음. thead 등의 구분을 쓰지 않아도 기능적으로는 작동함. 그러나 스크립트와 구분할때 사용하는것이 용이함.
셀 병합 속성은 rowspan세로병합, colaspan가로병합. ex) th colspan="2" / 같이 쓰는 스타일로는 border, border-width, border-style(실선,점선), border-color, border-collapse(셀테두리 1줄)

title태그
지정해둔 값에 마우스를 올렸을시 툴팁이 나옴. ex) title="이 글자가 나옵니다."
세련된 툴팁을 사용 하고 싶을시 ::after와 :hover 기능을 통해서 직접 스타일을 설정할 수 있음. 개념은 에프터를 통해서 박스 스타일을 미리설정하여 투명도를 0으로 설정하여 보이지 않게 설정. 그 후 호버시 투명도를 1로 설정하여 전시.

속성 선택자 표적방법
태그의 속성설정이 되었을때 특정 속성이 적용된 태그만 표적적용. a href="내 사진.png" 태그를 표적시
[속성] : 태그[속성]인 모두를 표적
a[href]
[속성="값"] : 속성값을 넣은 태그를 정확히 표적
a[href="내 사진.png"]
[속성^="값"] : 값으로 시작하는 태그를 표적
a[href^="내 사진"]
[속성$="값"] : 값으로 끝나는 태그를 표적
a[href=".png"]
[속성*="값"] : 값이 포함된 모두를 표적
a[href="pn"]
[속성~="값"] : 값이 독립적 사용된곳을 표적
a[href="사진.png"]

가상 클래스 선택자 :hover, :active, :focus, nth-child(n)
hover: 마우스를 올렸을때 사용. color 와 textdecoration 혹은 scale을 주로 같이 사용.
active: 마우스로 클릭하고있을떄. scale을 주로 매칭
focus: 클릭이나 탭키로 활성화 되었을때. 통상 outline: none 후에 border-color로 강조
nth-child(n): 같은 부모안에서 n자리에 들어가는 자식 선택. n설정으로는 2: 2번째 자식, 2n: 2번째 자식마다, odd/even: 홀수/짝수

우선순위 !important > 인라인 > ID > Class > 태그 선택자
css상 color: red !important;(스타일에서 주요도) > span style="color: red;"(인라인 직접 설정) > #id > .class > main

inherit; 부모의 속성 상속받기
부모로 부터 상속받지 않는 백그라운컬러, 보더 등과같은 박스단위를 꾸며주는 속성을 border: inherit; 으로 부모의 보더값을 받아옴.

box-sizing: content-box; / border-box;
content-box; 는 표준 요소. 즉 너비 높이의 설정이 콘텐츠박스 설정 후 보더값 패딩값 마진값 추가 적용
border-box; 는 사용자 설정. 마진까지 포함한 최종값을 결정. 마진값과 패딩값을 빼고 남은부분이 컨텐츠

position의 사용법
static; 기본상태
relative; 기준잡기 및 위치이동 보통 부모에 적용하여 자식의 기준점역할
absolute; 기준으로부터 정밀한 위치 조절. 주로 툴팁등에 사용
fixed; 상단 및 하단 고정에 사용. 맨위로 버튼 주로사용
sticky; 반고정상태. 부모영역에서만 고정되어 따라오다 부모의 박스영역에서 벗어날시 안따라옴.

플렉스 부모 사용 속성
flex-direction(정렬축 결정) : row(가로정렬: 기본값), column(세로정렬)
flex-wrap(줄바꿈) : nowrap(줄바꿈 없음 오버플로우: 기본값), wrap(너비 넘어갈시 자동 줄바꿈 허용)
flex-flow(통합버전): direction, wrap
justify-content(주축 정렬) : flex-start, flex-end, center, space-between, space-around, space-evenly
align-items(교차축 정렬): flex-start, flex-end, center, baseline, stretch(기본값)
align-content(여러줄 교차축 정렬): flex-start, flex-end, center, space-between, space-around, stretch(기본값)

플렉스 자식속성
flex-grow: 0 초기값. 남는 공간을 비율대로 분배
flex-shrink: 1 초기값. 부족시 줄어드는 비율
flex-basis: 길이px 크기설정 그로우와 슈린크의 연산기준값. auto설정시 height와 width의 값으로 됨. 연산은 적용
flex: grow shrink basis;
align-self: auto, flex-start, flex-end, center, baseline, stretch
order: 아이템의 순서 재배열. 주로 모바일 및 특정환경에서 구조순서를 바꿀때 사용.

transition 사용법
변화에 자연스러움을 줄 수 있는 속성. transition: 속성 지속시간 타이밍 지연시간;
지속시간: 변화에 몇초 걸릴지 0.3s등.
타이밍: 가속도설정 ease(기본값 천-빠-천) linear(등속) ease-in(천천히시작) ease-out(천천히끝) eaes-in-out(천빠천 더 부드럽)

meta : 메타데이터 / 웹페이지를 어떻게 해석하여 보여주면 좋을지 알려줌
meta charset="UTF-8" : 이페이지의 인코딩 방식을 UTF-8로 하겠다는뜻. 기본값.
meta name="viewport" content="width=device-width, initial-scale=1.0" : 이태그는 뷰포트(보이는 영역) 설정용이다의뜻, 너비는 디바이스너비, 최초 스케일은 100퍼비율
ㄴ 사실상 모바일 대응용으로 사용하며 이제는 서두에 기본적으로 들어가 줘야하는 부분.

@media 반응형 웹사이트 / 브레이크 포인트
미디어는 통상 모바일 혹은 태블릿과같은 디스플레이가 pc보다 비교적 작은곳에서 웹사이트를 이용 할 때 가독성을 높이기 위해 사용되는 규칙이다.
같이 사용되는 속성으로는 max-width, min-width를 주로사용. 최대넓이와 최소넓이를 설정하여, 조건부로 해석.
ex) max-width: 800px / 최대 넓이가 800px일때= 800px보다 적은 넓이 값에서~ 로 해석

브레이크 포인트란, 디바이스 종류에 따른 통상적인 변경점.
모바일: max-width:480 or 576px, 태블릿: min-width 768px, 데스크탑: min-width: 1024 or 1200px 울트라와이드: min-width 1440px이상
